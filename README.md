# Mr. Goli Soda — Franchise Finance (Flask + Supabase + Render)

Shared web app to track franchise marketing/prep expenses — **franchise-wise → state-wise → overall**, budget flags (yellow ≥ ₹2L, red > ₹3L), add-anytime categories, Excel/CSV import, edit/delete, search, month filter, bills-pending, CSV export. One link, one login, shared live data.

**Stack:** Python **Flask** + **PostgreSQL (Supabase)**, served by **gunicorn** on **Render**. Frontend is plain HTML/CSS/JS in `static/`.

```
app.py            Flask app + REST API
db.py             Postgres connection pool, schema, seed data
requirements.txt  Python dependencies
Procfile          gunicorn start command
render.yaml       Render Blueprint (web service + env vars)
static/
  index.html      the whole UI
  logo.png        Mr Goli Soda logo
  vendor/xlsx...  Excel reader (bundled, works offline)
```

## Step 1 — Create the database on Supabase
1. Go to **supabase.com**, sign in, **New project**. Pick a name + a strong **database password** (save it).
2. Wait for it to finish provisioning.
3. Open **Project Settings → Database → Connection string**.
4. Choose the **Connection pooling** string (recommended for Render), **Session** mode. It looks like:
   `postgresql://postgres.<ref>:[YOUR-PASSWORD]@aws-0-...pooler.supabase.com:5432/postgres`
5. Replace `[YOUR-PASSWORD]` with the database password from step 1. **This whole string is your `DATABASE_URL`.**

> Use the **pooler** string (not the raw `db.<ref>.supabase.co` one) — Render connects to it reliably. SSL is added automatically by the app.

The app creates its tables and seeds the 29 franchises + 6 categories on first start. No SQL to run by hand.

## Step 2 — Deploy on Render
1. Put this folder in a **private GitHub repo**.
2. Render → **New + → Blueprint** → pick the repo (it reads `render.yaml`).
3. When prompted, paste your Supabase string into **`DATABASE_URL`**. (`APP_PASSWORD`, `ADMIN_PASSWORD`, `SECRET_KEY` are already set.)
4. **Apply**. After ~3–5 min you get a link like `https://gac-finance.onrender.com`.

## Passwords / env vars
| Variable | Meaning | Set where |
|---|---|---|
| `DATABASE_URL` | Supabase connection string | You paste it on Render |
| `APP_PASSWORD` | Team login (`SAFC@123`) | Baked in `render.yaml` |
| `ADMIN_PASSWORD` | Needed for "Replace all" (`Begum@123`) | Baked in `render.yaml` |
| `SECRET_KEY` | Signs login sessions | Auto-generated by Render |

Change a password by editing `render.yaml` (or the Render dashboard) and redeploying.

## Backups
- **Supabase** keeps automatic daily backups (and point-in-time on paid plans).
- **In-app:** the **Export CSV** button downloads a full copy anytime — use it as a habit.
